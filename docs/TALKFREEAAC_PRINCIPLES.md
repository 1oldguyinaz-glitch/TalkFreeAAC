# TalkFreeAAC Engineering Principles

1. Communication is always free.
2. The user's sentence is never lost unless they intentionally clear it.
3. No action should require unnecessary taps.
4. The interface adapts to the device, not the other way around.
5. The first screen should enable meaningful conversation within minutes.
6. Every feature must reduce communication friction.
7. Relationship language matters as much as request language.
8. The home board should help users express needs, emotions, relationships, curiosity, and personality.
9. Core communication must work offline and remain local-first by default.
10. Professional insights are optional; communication itself is never paywalled.
