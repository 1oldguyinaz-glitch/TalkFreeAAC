# TalkFreeAAC V8.1 — Full 10,000-Entry Fitzgerald Routing

## Scope

V8.1 enriches every entry in `engine/language/languageDatabaseV5_27_10000.json` with a deterministic six-column Fitzgerald motor-plan route.

No entry is left unclassified. The database remains lazy loaded and bucket gated; this patch does not dump the full language inventory onto the child board.

## Six-column route

1. Initiators, pronouns, and questions
2. Actions and verbs
3. Modifiers, negations, connectors, and prepositions
4. Possessives and determiners
5. Describers, emotions, states, and adjectives
6. Targets and noun buckets

## Routing method

The builder uses a fixed, reproducible priority order:

1. Always-active safety/control overrides
2. Multi-role grammatical overrides
3. Pronoun, question, determiner, modifier, and action lexicons
4. Phrase/template prefix analysis
5. Existing V7.1 semantic concept-graph categories and buckets
6. Safe target fallback

The classification can be regenerated by running:

```bash
node engine/language/buildFitzgeraldRoutingV8_1.mjs
```

## Motor-plan safeguards

- Page size is fixed at 10 entries per route bucket.
- Each `(column, bucket, page, slot)` coordinate is unique.
- Stable ordering prioritizes communication stage, utility score, prediction weight, and original database order.
- Yes, No, Help, Stop, and Clear remain active in States 1–6.
- Known verbs retain their existing Column 3 grammar-profile links.
- Final targets reset the state machine to State 1.
